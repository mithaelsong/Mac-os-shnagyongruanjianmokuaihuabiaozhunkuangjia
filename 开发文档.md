## 功能01：主程序启动

### 用途
应用程序入口文件。初始化框架所有组件，按有序阶段启动，串联功能02~30全部能力。

### 核心类

| 类名 | 类型 | 说明 |
|------|------|------|
| `XRZApplication` | public final class | 应用主入口单例，管理29阶段启动流程 |
| `LaunchPhase` | public enum | 30个启动阶段枚举（29功能+completed） |
| `LaunchResult` | public enum | 启动结果：success / partialFailure / criticalFailure |
| `XRZAppDelegate` | public final class | NSApplicationDelegate |

### 公开API

```swift
public static let shared: XRZApplication
@discardableResult public func start(pluginDirectory: URL? = nil) -> LaunchResult
public func shutdown()
public func phasePassed(_ phase: LaunchPhase) -> Bool?
public var launchPhase: LaunchPhase  // 当前阶段
public var isRunning: Bool           // 运行状态
public var mainWindow: NSWindow?     // 主窗口
public var launchResult: LaunchResult // 启动结果
```

## 功能02：初始化日志系统

### 用途
统一的日志记录系统，支持多级别日志、多输出目标、文件轮转、线程安全。

### 核心类/结构体

| 类名 | 类型 | 说明 |
|------|------|------|
| `LogSystem` | class | 日志系统单例，initialize方法启动 |
| `ModuleLogger` | class | 模块级日志记录器，每个模块一个实例 |
| `LogLevel` | enum | 日志级别：debug/info/warning/error/fatal |
| `LogEntry` | struct | 单条日志记录，含时间戳/级别/消息/分类 |
| `ConsoleLogOutput` | class | 控制台输出目标 |
| `FileLogOutput` | class | 文件输出目标，支持轮转和清理 |
| `LogOutput` | protocol | 日志输出协议，可自定义输出目标 |

### 公开API

```swift
// LogSystem
LogSystem.shared.initialize()
LogSystem.shared.addOutput(_ output: LogOutput)
LogSystem.shared.flush()

// ModuleLogger
let logger = ModuleLogger(category: "MyModule")
logger.debug("消息")
logger.info("消息")
logger.warning("警告")
logger.error("错误")
logger.fatal("致命错误")
```

### 关键功能点

| 功能 | 说明 |
|------|------|
| 多级别 | debug/info/warning/error/fatal |
| 多输出目标 | 控制台/文件/自定义 |
| 文件轮转 | 超过大小自动轮转，可配置保留数量 |
| 线程安全 | os_unfair_lock保护所有写入 |
| 日志清理 | 自动清理过期日志文件 |
| 模块分类 | 每个ModuleLogger有独立category标签 |

## 功能05：按顺序加载模块

### 用途
按依赖关系的拓扑排序依次加载模块，支持循环依赖检测。

### 核心类/结构体

| 类名 | 类型 | 说明 |
|------|------|------|
| `ModuleLoader` | class | 加载器，依赖ModuleRegistry/EventBus |
| `DependencyResolver` | class | 依赖解析器，拓扑排序 |
| `ModuleError` | enum | 加载错误类型 |
| `ModuleLoadResult` | enum | 加载结果（success/failure/alreadyLoaded） |
| `XRZModule` | protocol | 模块入口协议 |

### 公开API

```swift
// ModuleLoader
let loader = ModuleLoader(registry: registry, eventBus: eventBus, logger: logger)
loader.load(module: scannedModule, metadata: metadata) -> ModuleLoadResult
loader.loadModules(modules: [ScannedModule]) -> [String: ModuleLoadResult]
loader.loadAll() -> [ModuleLoadResult]
loader.unload(name: String) -> ModuleLoadResult
loader.unloadAll() -> [ModuleLoadResult]

// DependencyResolver
DependencyResolver.resolveLoadOrder(modules: [String: [String]]) throws -> [String]
```

### 关键功能点

| 功能 | 说明 |
|------|------|
| 拓扑排序 | Kahn算法，按依赖顺序确定加载序列 |
| 循环依赖检测 | throw错误，避免死锁 |
| 幂等加载 | 已加载模块返回.alreadyLoaded |
| 依赖缺失 | 自动跳过并记录错误 |

## 功能07：处理模块加载失败

### 用途
5种加载失败类型的处理策略：重试/降级/默认配置/中止/放弃，含指数退避重试。

### 核心类/结构体

| 类名 | 类型 | 说明 |
|------|------|------|
| `ModuleFailureHandler` | class | 失败处理器 |
| `ModuleFailureType` | enum | 5种失败类型 |
| `ModuleFailureResolution` | enum | 处理策略决议 |

### 公开API

```swift
let handler = ModuleFailureHandler()
handler.handle(.dependencyMissing(module: "KLine", dependency: "DataFeed")) -> ModuleFailureResolution
handler.handle(.circularDependency(path: [...])) -> ModuleFailureResolution
handler.handle(.versionIncompatible(module: "KLine", required: "2.0", actual: "1.0")) -> ModuleFailureResolution
handler.handle(.configurationError(module: "KLine", reason: "missing key")) -> ModuleFailureResolution
handler.handle(.loadTimeout(module: "KLine", duration: 30.0)) -> ModuleFailureResolution
handler.canRetry(module: "KLine") -> Bool
handler.reset(module: "KLine")
handler.resetAll()
```

### 关键功能点

| 功能 | 说明 |
|------|------|
| 5种失败类型 | 依赖缺失/循环依赖/版本不兼容/配置错误/加载超时 |
| 4种策略 | retry/downgrade/useDefault/abort |
| 指数退避 | baseDelay × 2^(attempt-1) |
| 放弃机制 | 超过maxRetries(3)自动giveUp |
| 状态重置 | reset/resetAll清空重试记录 |

## 功能12：模块热替换

### 用途
运行时替换已加载模块为新版本，6阶段流程+完整回滚。

### 核心类/结构体

| 类名 | 类型 | 说明 |
|------|------|------|
| `ModuleHotSwapper` | class | 热替换器 |
| `HotSwapResult` | enum | 热替换结果 |
| `HotSwapFailureReason` | enum | 失败原因 |
| `ModuleStateSavable` | protocol | 状态可保存协议 |
| `ModuleStateSnapshot` | struct | 状态快照 |

### 公开API

```swift
let swapper = ModuleHotSwapper(registry: registry, loader: loader, unloader: unloader, eventBus: eventBus)
swapper.hotSwap(moduleName: "KLine", with: newBundlePath) -> HotSwapResult
swapper.hotSwapBatch([(String, URL)]) -> [HotSwapResult]
swapper.registerSavableModule(moduleName: "KLine", savable: savableInstance)
```

### 6阶段流程

| 阶段 | 说明 |
|------|------|
| 1. save | 保存模块状态（通过ModuleStateSavable） |
| 2. scan | 扫描新bundle，验证签名 |
| 3. unload | 卸载旧模块 |
| 4. load | 加载新模块 |
| 5. start | 启动新模块 |
| 6. restore | 恢复保存的状态 |

任何阶段失败自动触发回滚。
